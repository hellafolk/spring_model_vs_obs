# Model vs. Observations of CO₂ in the Tropical Pacific Marine Boundary Layer

**Author:** A. McAuliffe  
**Mentors:** J. Peischl, D. Munro  

---

## Project Description

This project investigates whether systematic offsets between in-situ atmospheric CO₂ observations and model reference products are unique to a single observing platform or common across multiple independent platforms in the tropical Pacific marine boundary layer (MBL). Three observing platforms, two research vessels and the NASA ATom DC-8 aircraft, are compared against three atmospheric CO₂ reference products (NOAA Global MBL, NOAA Pacific MBL, and CarbonTracker CT2022) over the period 2016–2018. Remote sensing data (SST, chlorophyll-a, wind speed, surface pressure) are time-matched to observations to explore potential drivers of the observed offsets. A random forest regression model is trained to predict CO₂ from environmental and geographic features.

## Data

Data used in this project are not stored in this repository due to file size. All remote sensing and model data are downloaded automatically by the notebook. Observational data (ATom, TF5, POC) must be obtained separately and placed in the correct directories as described below.

### Data that download automatically when the notebook is run:
- **OISST SST** — NOAA ERDDAP
- **MODIS-Aqua Chlorophyll-a** — NASA CoastWatch ERDDAP
- **ERA5 Wind + Pressure** — Copernicus Climate Data Store (requires free CDS account at https://cds.climate.copernicus.eu)
- **CarbonTracker CT2022** — NOAA GML

### Data that must be obtained and placed manually:
- **ATom MER-1HZ files** — available from the NASA ESPO archive at https://espo.nasa.gov/atom
  - Place in: `/Volumes/USBDRIVE/finalproj/data/observations/atom/`
- **TF5 Shipboard Picarro** — available from NOAA GML
  - Place in: `/Volumes/USBDRIVE/finalproj/data/observations/ship/`
- **POC Shipboard Flask** — available from NOAA GML at https://gml.noaa.gov/dv/data/
  - Place in: `/Volumes/USBDRIVE/finalproj/data/observations/ship/`
- **NOAA MBL Reference products** — available from https://gml.noaa.gov/ccgg/mbl/
  - Place Global MBL in: `/Volumes/USBDRIVE/finalproj/data/mbl/mbl_global/`
  - Place Pacific MBL in: `/Volumes/USBDRIVE/finalproj/data/mbl/mbl_pacific/`

## How to Run

### Requirements

Install the required Python packages:

```bash
pip install numpy pandas netCDF4 matplotlib cartopy scipy scikit-learn imageio cdsapi
```

You will also need a free Copernicus CDS account for the ERA5 download. Once registered, add your API key to `~/.cdsapirc`:

```
url: https://cds.climate.copernicus.eu/api
key: YOUR-API-KEY-HERE
```

### Running the notebook

1. Clone this repository
2. Place observational data files in the directories listed above
3. Open `notebooks/McAuliffe_tropical_mbl_co2.ipynb` in Jupyter
4. Run all cells from top to bottom (`Kernel → Restart & Run All`)

The notebook will:
- Create all output directories automatically
- Download SST, chlorophyll, ERA5, and CarbonTracker data automatically
- Load and filter all observational data
- Compute offsets from all three reference products
- Match remote sensing data to observations
- Generate all figures and save to `outputs/figures/`
- Train the random forest and output results

**Note:** The full notebook takes approximately 20–30 minutes to run due to data downloading and matching loops.
